Lesson 1 - Basic bzr

First let's establish who you are. (Replace the name and email with your name and email.)

bzr whoami
"John Doe <john.doe@gmail.com>"

Now if you type bzr whoami - you should get what you typed in.

We're going to need a directory: 

mkdir test_dir

Now cd into that folder. Then, to say that you want to use bzr in it:

bzr init

Let's create a few subdirectories & files:

mkdir sub_dir
touch test1.txt test2.txt test3.txt sub_dir/test4.txt

Next, you want bzr to keep track of all those files and the directory. The command bzr add * will recursively add every file and directory within your current directory

bzr add *

Note: You'll have to use bzr add whenever you add a new file/directory.

Next, take a snapshot of your files by committing them to your branch. Add a message to explain why you made the commit:

bzr commit -m "Initial import"

So, you have four files, one in a subdirectory, called sub_dir. Open test1.txt; copy and paste (without quotes) this line into the file: "Testing 1, 2, 3"; save it and go back to the terminal screen.

Now when you type bzr diff you should get something like this:

=== modified file 'test1.txt'
--- test1.txt   2008-06-04 15:57:45 +0000
+++ test1.txt   2008-06-04 16:01:37 +0000
@@ -0,0 +1,1 @@
+Testing 1, 2, 3

The only really important thing there is the bottom line; it says that there is one difference from the last commit and the current version of the file, you added a line, "Testing 1, 2, 3".

Now let's commit that revision, essentially we'll be taking a new snapshot of the files.

bzr commit -m "Added first line to test1.txt"

There's actually a way to view the log; which shows each revision.

bzr log

You should get a result like this:

------------------------------------------------------------
revno: 1
committer: Daniel Folsom <danielfolsom@gmail.com>
branch nick: bzr.testdirectory
timestamp: Thu 2008-06-05 10:02:02 -0400
message:
  Started repository
