Lesson 5 - Launchpad

Launchpad is one of the main reasons why we have been able to make this tutoral and the others in the series. Launchpad is a software hosting and developing website. It uses bzr as its revision control system.

In order to use Launchpad, you will need to create an account.

You'll also need an SSH key. Make sure you have all your updates installed, then in the terminal screen, type:

ssh-keygen -t dsa

Do not enter a password or any other location than the default provided for the .pub file. Once that process is complete, find your public ssh key (in the file id_dsa.pub within your .ssh directory) and copy the contents of the file to your Launchpad account. There is a place on your user screen in Launchpad to "Update your SSH keys". Simply paste the content of the .pub file there and save.

Once you have an account and your SSH key you must learn some very important bzr commands to use with launchpad.

Let's push up your folder to launchpad.

bzr push bzr+ssh://yourusername@bazaar.launchpad.net/~yourusername/+junk/myproject

+Note: The +junk denotes that the folder is not associated with any particular project.

By pushing up - you have given the world the ability to download your folder. All anyone would need to type is:

bzr branch http://bazaar.launchpad.net/~yourusername/+junk/myproject

And the files would be downloaded to their computer. What this is especially useful for is team collaboration. You can have multiple people working on a folder, and all they would need to do is push & pull changes back and forth. Speaking of pulling and pushing: make any change to a document in your folder and commit it. Now you probably want to push that change up to Launchpad, right? When you put in the horrendously long http://baz... line, your computer actually memorized that location for your folder, so, now, all you need to do is:

bzr push

And you will update your launchpad folder. Now, say someone else has made a change to your file; you have to pull those changes from Launchpad.

bzr pull

Again, when you first entered the location, your computer memorized it; so you don't need to type bzr pull http://bazaar...

But obviously there can be problems - if two people make changes at the same time you will have to merge in order for both revisions to make it back up to Launchpad. After trying to push if you recieve a message when pushing that the branches have diverged then you need to use bzr merge.

bzr merge

After sucessfully merging then you need to re-commit the code.

bzr commit -m "merged"

You then can push it up if you still recieve a message do the process over again until it pushes up to Launchpad.

Sometimes people decide to shutdown there computer before they finishing pushing. This creates a lock on the server so that the entire system does not get screwed up. The only way to get out of this is to break the lock.

bzr break-lock http://bazaar.launchpad.net/~yourusername/+junk/myproject

This will allow you to get access to the server again, you now would be able to commit, merge and push back up to launchpad.

Here's a short little recap of some of the commands:

BZR Commands
bzr commit -m "[What you did]"Commits a revision to the bzr log. You can use bzr commit [file] -m "[What you did]" to commit one file only.
bzr logGives a log of all of the saved revisions
bzr revert --revision [revision#]Reverts an edit to the previous point in the log. You can use bzr revert [file] --revision [revision#] to revert one file only.
bzr diffShows the difference between the current version and the current logged version.
bzr pushGives the folder's changes to launchpad.
bzr branch http://bazaar.launchpad.net/~john.doe/+junk/myprojectGets the folder from launchpad.
bzr add *Adds new files to the bzr repository. Must be done the first time you have files, and every time you add a new file
